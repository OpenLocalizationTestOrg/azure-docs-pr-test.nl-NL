# <a name="understanding-azure-virtual-machine-usage"></a>Informatie over het gebruik van de virtuele machine van Azure

Voor sommige afbeeldingen in de Azure-galerie, wordt het afbeeldingstype ingevuld in het veld aanvullende informatie. Hierdoor kunnen gebruikers om te begrijpen en te houden wat ze voor hun virtuele Machine hebt geïmplementeerd. De waarden die worden ingevuld in dit veld op basis van de installatiekopie die u hebt geïmplementeerd, zijn de volgende:

  - SQL Server 14 Preview op Windows Server 2012 R2 Preview 
  - SUSE
  - SUSE Premium

  - Red Hat
  - Red Hat voor SAP Business-toepassingen     
  - Red Hat voor SAP HANA

  - WindowsServer BYOL

## <a name="service-type"></a>Servicetype
Het veld service type in het veld aanvullende informatie overeenkomt met de exacte VM-grootte die u hebt geïmplementeerd. Premium-opslag virtuele machines (op SSD gebaseerd) en niet voor de premium-opslag (HDD-indeling) virtuele machines zijn dezelfde prijs. Als u een SSD-grootte, zoals standaard implementeert\_DS2\_v2, ziet u de grootte van de niet-SSD ("standaard\_D2\_v2 VM') in de kolom subcategorie Meter en de SSD-grootte (' standaard\_DS2\_ v2') in het veld aanvullende informatie.

## <a name="virtual-machine-usage-faq"></a>Gebruik van virtuele machines Veelgestelde vragen
### <a name="what-resources-are-charged-when-deploying-a-vm"></a>Welke bronnen worden in rekening gebracht bij het implementeren van een virtuele machine?    
Virtuele machines verkrijgen kosten voor de virtuele machine zelf, premium software op de virtuele machine, de opslagschijf account\managed die zijn gekoppeld aan de virtuele machine wordt uitgevoerd en de VPN-bandbreedte worden overgedragen van de virtuele machine.
### <a name="how-can-i-tell-if-a-vm-is-using-azure-hybrid-benefit-in-the-usage-csv"></a>Hoe kan ik zien of een virtuele machine Azure hybride voordeel wordt gebruikt in de CSV-gebruik
Als u met behulp van implementeert de [Azure hybride voordeel](https://azure.microsoft.com/pricing/hybrid-benefit/), u in rekening worden gebracht de frequentie van niet-Windows virtuele machine omdat u uw eigen licentie naar de cloud meebrengen. In uw factuur, kunt u onderscheiden welke Resource Manager VMs Azure hybride voordeel worden uitgevoerd, omdat ze ' Windows\_Server BYOL ' of ' Windows\_Client BYOL ' in de kolom ImageType.
### <a name="how-are-basic-vs-standard-vm-types-differentiated-in-the-usage-csv"></a>Hoe worden Basic vs. Standaard VM-typen differentiated CSV gebruik?
Zowel Basic en Standard A-serie van virtuele machines worden aangeboden. Als u een eenvoudige virtuele machine, in de categorie van het Sub Meter implementeren is de tekenreeks 'Basic'. Als u een standaard A-serie VM implementeert, klikt u vervolgens de VM-grootte wordt weergegeven als "A1-VM" omdat standaard de standaardinstelling is. Zie voor meer informatie over de verschillen tussen Basic en Standard, de [pagina met prijzen](https://azure.microsoft.com/pricing/details/virtual-machines/).
### <a name="what-are-extrasmall-small-medium-large-and-extralarge-sizes"></a>Wat zijn extra klein, klein, normaal, groot en extra groot grootten?
ExtraSmall - zijn, zijn de verouderde namen voor standaard\_A0 – Standard\_A4. U ziet in de klassieke VM gebruiksrecords, deze overeenkomst gebruikt als u deze formaten hebt geïmplementeerd.
### <a name="what-is-the-difference-between-meter-region-and-resource-location"></a>Wat is het verschil tussen Meter regio en Resourcelocatie?
De Meter regio is gekoppeld aan de meter. Voor sommige Azure-services die gebruikmaken van één prijs voor alle regio's, kan het veld regio van de Meter niet leeg zijn. Aangezien de virtuele machines hebben de prijzen per regio speciale voor virtuele Machines, wordt echter in dit veld ingevuld. Op deze manier is de locatie van de bron voor virtuele Machines van de locatie waar de virtuele machine is geïmplementeerd. De Azure-regio in beide velden zijn hetzelfde, hoewel ze mogelijk een andere tekenreeks conventie voor de regionaam hebben.
### <a name="why-is-the-imagetype-value-blank-in-the-additional-info-field"></a>Waarom is de waarde ImageType leeg in het veld aanvullende informatie?
Het veld ImageType wordt alleen ingevuld voor een subset van afbeeldingen. Als u een van de bovenstaande afbeeldingen niet hebt geïmplementeerd, is de ImageType is leeg.
### <a name="why-is-the-vmname-blank-in-the-additional-info"></a>Waarom is de VMName leeg in de aanvullende gegevens?
De VMName wordt alleen gevuld in het veld aanvullende informatie voor virtuele machines in een schaalset. Het veld InstanceID bevat de naam van de virtuele machine voor niet-virtuele machines schaalset.
### <a name="what-does-computehr-mean-in-the-usagetype-field-in-the-additional-info"></a>Wat betekent ComputeHR in het veld UsageType in de aanvullende gegevens?
ComputeHR staat voor Compute uur waarmee de gebeurtenis gebruik voor de onderliggende infrasturcture kosten. Als de UsageType ComputeHR\_SW, de gebeurtenis gebruik vertegenwoordigt de premium-software-kosten voor de virtuele machine.
### <a name="how-do-i-know-if-i-am-charged-for-premium-software"></a>Hoe weet als ik kosten in rekening voor premium-software gebracht ben?
Wanneer ontdekken welke VM-installatiekopie beste past bij uw behoeften, moet u zien wat de [Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps/category/compute). De installatiekopie heeft de snelheid van software-abonnement. Als u 'Gratis' voor de frequentie waarmee ziet, is er zonder extra kosten voor de software. 
### <a name="what-is-the-difference-between-microsoftclassiccompute-and-microsoftcompute-in-the-consumed-service"></a>Wat is het verschil tussen Microsoft.ClassicCompute en Microsoft.Compute in de service verbruikt?
Microsoft.ClassicCompute vertegenwoordigt klassieke resources via de Azure Service Manager wordt geïmplementeerd. Als u via de Resource Manager implementeert, wordt in de verbruikte service Microsoft.Compute gevuld. Meer informatie over de [Azure-implementatiemodellen](/azure-resource-manager/resource-manager-deployment-model.md).
### <a name="why-is-the-instanceid-field-blank-for-my-virtual-machine-usage"></a>Waarom is het exemplaar-id-veld leeg laten voor mijn gebruik van virtuele machines?
Als u via het klassieke implementatiemodel implementeert, is de exemplaar-id-tekenreeks niet beschikbaar.
### <a name="why-are-the-tags-for-my-vms-not-flowing-to-the-usage-details"></a>Waarom zijn de labels voor mijn VM's die niet aan de verbruiksgegevens stromende?
Labels alleen stromen voor u het gebruik van CSV voor Resource Manager virtuele machines alleen. Klassieke resourcelabels zijn niet beschikbaar in de details van het gebruik.
### <a name="how-can-the-consumed-quantity-be-more-than-24-hours-one-day"></a>Hoe kan de hoeveelheid verbruikte meer dan 24 uur één dag zijn?
In het klassieke model, wordt de facturering voor bronnen op het niveau van de Cloudservice geaggregeerd. Als u meer dan één virtuele machine in een Cloudservice die gebruikmaakt van de dezelfde facturering meter hebt, wordt uw gebruik samen geaggregeerd. Via Resource Manager geïmplementeerde virtuele machines worden gefactureerd op het niveau van de virtuele machine, zodat de aggregatie is niet van toepassing.
### <a name="why-is-pricing-not-available-for-dsfsgsls-sizes-on-the-pricing-page"></a>Waarom is prijzen niet beschikbaar voor DS/FS/GS/LS grootten op de pagina met prijzen?
Premium-opslag kunnen virtuele machines worden gefactureerd op dezelfde snelheid als niet-premium-opslag kunnen virtuele machines. Alleen de opslagkosten verschillen. Ga naar de [opslag-pagina met prijzen](https://azure.microsoft.com/pricing/details/storage/unmanaged-disks/) voor meer informatie.
