# <a name="create-network-security-groups-using-the-azure-cli"></a>Netwerk maken met de Azure CLI-beveiligingsgroepen

[!INCLUDE [virtual-networks-create-nsg-selectors-arm-include](../../includes/virtual-networks-create-nsg-selectors-arm-include.md)]

[!INCLUDE [virtual-networks-create-nsg-intro-include](../../includes/virtual-networks-create-nsg-intro-include.md)]

[!INCLUDE [virtual-networks-create-nsg-scenario-include](../../includes/virtual-networks-create-nsg-scenario-include.md)]

De volgende voorbeeldopdrachten van Azure CLI verwachten dat een bestaande eenvoudige omgeving op basis van het voorgaande scenario. 

## <a name="create-the-nsg-for-the-frontend-subnet"></a>Maken van het NSG voor de `FrontEnd` subnet

Maken van een NSG met de naam *NSG-FrontEnd* op basis van het voorgaande scenario, volgt u de volgende stappen.

1. Als u dit nog niet hebt nog installeren en configureren van de meest recente [Azure CLI 2.0](/cli/azure/install-az-cli2) en meld u aan op een Azure-account met [az aanmelding](/cli/azure/#login). 

2. Maakt een NSG met behulp van de [az netwerk nsg maken](/cli/azure/network/nsg#create) opdracht. 

    ```azurecli
    az network nsg create \
    --resource-group testrg \
    --name NSG-FrontEnd \
    --location centralus 
    ```

    Parameters:
   
   * `--resource-group`: De naam van de resourcegroep waar de NSG wordt gemaakt. In ons scenario *TestRG*.
   * `--location`: De azure-regio waar de nieuwe NSG gemaakt. In ons scenario *westus*.
   * `--name`: De naam voor de nieuwe NSG. In ons scenario *NSG-FrontEnd*.

    De verwachte uitvoer is erg een stukje informatie, waaronder een lijst met alle standaardregels. Het volgende voorbeeld ziet u de standaardregels die met een filter JMESPATH query met de `table` uitvoerindeling:

    ```azurecli
    az network nsg show \
    -g testrg \
    -n nsg-frontend \
    --query 'defaultSecurityRules[].{Access:access,Desc:description,DestPortRange:destinationPortRange,Direction:direction,Priority:priority}' \
    -o table
    ```
   
   Uitvoer:

        Access    Desc                                                    DestPortRange    Direction      Priority
        
        Allow     Allow inbound traffic from all VMs in VNET              *                Inbound           65000
        Allow     Allow inbound traffic from azure load balancer          *                Inbound           65001
        Deny      Deny all inbound traffic                                *                Inbound           65500
        Allow     Allow outbound traffic from all VMs to all VMs in VNET  *                Outbound          65000
        Allow     Allow outbound traffic from all VMs to Internet         *                Outbound          65001
        Deny      Deny all outbound traffic                               *                Outbound          65500



3. Maak een regel waarmee toegang tot poort 3389 (RDP) van het Internet met de [az netwerk nsg regel maken](/cli/azure/network/nsg/rule#create) opdracht.

    > [!NOTE]
    > Afhankelijk van de shell u gebruikt, moet u mogelijk wijzigen de `*` teken in de volgende om het argument vóór uitvoering Vouw argumenten.
   
    ```azurecli
    az network nsg rule create \
    --resource-group testrg \
    --nsg-name NSG-FrontEnd \
    --name rdp-rule \
    --access Allow \
    --protocol Tcp \
    --direction Inbound \
    --priority 100 \
    --source-address-prefix Internet \
    --source-port-range "*" \
    --destination-address-prefix "*" \
    --destination-port-range 3389
    ```
   
    Verwachte uitvoer:
   
    ```json
    {
        "access": "Allow",
        "description": null,
        "destinationAddressPrefix": "*",
        "destinationPortRange": "3389",
        "direction": "Inbound",
        "etag": "W/\"<guid>\"",
        "id": "/subscriptions/<guid>/resourceGroups/testrg/providers/Microsoft.Network/networkSecurityGroups/NSG-FrontEnd/securityRules/rdp-rule",
        "name": "rdp-rule",
        "priority": 100,
        "protocol": "Tcp",
        "provisioningState": "Succeeded",
        "resourceGroup": "testrg",
        "sourceAddressPrefix": "Internet",
        "sourcePortRange": "*"
    }
    ```

    Parameters:

    * `--resource-group testrg`: De resourcegroep te gebruiken. Houd er rekening mee dat het is niet hoofdlettergevoelig.
    * `--nsg-name NSG-FrontEnd`: De naam van de NSG waarin de regel is gemaakt.
    * `--name rdp-rule`: De naam voor de nieuwe regel.
    * `--access Allow`: Toegangsniveau voor de regel (weigeren of toestaan).
    * `--protocol Tcp`: Protocol (Tcp, Udp of *).
    * `--direction Inbound`: De richting van de verbinding (inkomend of uitgaand).
    * `--priority 100`: De prioriteit voor de regel.
    * `--source-address-prefix Internet`: Bron-adresvoorvoegsel in CIDR- of met standaardlabels.
    * `--source-port-range "*"`: Poort of poortbereik van bron. De poort die de verbinding geopend.
    * `--destination-address-prefix "*"`: Voorvoegsel voor doeladres in CIDR- of met standaardlabels.
    * `--destination-port-range 3389`: Bestemming poort of poortbereik. De poort die de verbindingsaanvraag ontvangt.



4. Maak een regel die toegang tot poort 80 (HTTP) van het Internet is **az netwerk nsg regel maken** opdracht.

    ```azurecli
    --destination-port-range 80
    ```
   
    Verwachte putput:

## <a name="create-the-nsg-for-the-backend-subnet"></a>Maken van het NSG voor de `BackEnd` subnet
Maken van een NSG met de naam *NSG-back-end* op basis van het voorgaande scenario, volgt u de volgende stappen.

1. Maak de `NSG-BackEnd` NSG met **az netwerk nsg maken**.
   
    ```azurecli
    az network nsg create \
    --resource-group testrg \
    --name NSG-BackEnd \
    --location centralus
    ```
   
    Zoals in stap 2, voorgaande, is de verwachte uitvoer erg groot is, met inbegrip van standaardregels.
   
2. Maak een regel waarmee toegang tot poort 1433 (SQL) van de `FrontEnd` subnet met de **az netwerk nsg regel maken** opdracht.
   
    ```azurecli
    az network nsg rule create \
    --resource-group testrg \
    --nsg-name NSG-BackEnd \
    --name sql-rule \
    --access Allow \
    --protocol Tcp \
    --direction Inbound \
    --priority 100 \
    --source-address-prefix 192.168.1.0/24 \
    --source-port-range "*" \
    --destination-address-prefix "*" \
    --destination-port-range 1433
    ```
   
    Verwachte uitvoer:

    ```json  
    {
    "access": "Allow",
    "description": null,
    "destinationAddressPrefix": "*",
    "destinationPortRange": "1433",
    "direction": "Inbound",
    "etag": "W/\"<guid>\"",
    "id": "/subscriptions/<guid>/resourceGroups/testrg/providers/Microsoft.Network/networkSecurityGroups/NSG-BackEnd/securityRules/sql-rule",
    "name": "sql-rule",
    "priority": 100,
    "protocol": "Tcp",
    "provisioningState": "Succeeded",
    "resourceGroup": "testrg",
    "sourceAddressPrefix": "192.168.1.0/24",
    "sourcePortRange": "*"
    }
    ```

3. Maak een regel die de toegang aan het Internet via weigert de **az netwerk nsg regel maken** opdracht.
   
    ```azurecli
    az network nsg rule create \
    --resource-group testrg \
    --nsg-name NSG-BackEnd \
    --name web-rule \
    --access Deny \
    --protocol Tcp  \
    --direction Outbound  \
    --priority 200 \
    --source-address-prefix "*" \
    --source-port-range "*" \
    --destination-address-prefix "*" \
    --destination-port-range "*"
    ```
   
    Verwachte putput:
   
    ```json
    {
    "access": "Deny",
    "description": null,
    "destinationAddressPrefix": "*",
    "destinationPortRange": "*",
    "direction": "Outbound",
    "etag": "W/\"<guid>\"",
    "id": "/subscriptions/<guid>/resourceGroups/testrg/providers/Microsoft.Network/networkSecurityGroups/NSG-BackEnd/securityRules/web-rule",
    "name": "web-rule",
    "priority": 200,
    "protocol": "Tcp",
    "provisioningState": "Succeeded",
    "resourceGroup": "testrg",
    "sourceAddressPrefix": "*",
    "sourcePortRange": "*"
    }
    ```
