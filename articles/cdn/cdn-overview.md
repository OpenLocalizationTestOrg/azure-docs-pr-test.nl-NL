## <a name="how-it-works"></a><span data-ttu-id="cabb8-113">Hoe werkt het?</span><span class="sxs-lookup"><span data-stu-id="cabb8-113">How it works</span></span>
![Overzicht van CDN](./media/cdn-overview/cdn-overview.png)

1. <span data-ttu-id="cabb8-115">Een gebruiker (Els) gebruikt een URL met een speciale domeinnaam, zoals `<endpointname>.azureedge.net`, om een bestand (ook wel een asset genoemd) aan te vragen.</span><span class="sxs-lookup"><span data-stu-id="cabb8-115">A user (Alice) requests a file (also called an asset) using a URL with a special domain name, such as `<endpointname>.azureedge.net`.</span></span>  <span data-ttu-id="cabb8-116">De aanvraag wordt door DNS naar de best presterende POP-locatie (Point-of-Presence) gerouteerd.</span><span class="sxs-lookup"><span data-stu-id="cabb8-116">DNS routes the request to the best performing Point-of-Presence (POP) location.</span></span>  <span data-ttu-id="cabb8-117">Doorgaans is dit het POP dat zie geografisch gezien het dichtst bij de gebruiker bevindt.</span><span class="sxs-lookup"><span data-stu-id="cabb8-117">Usually this is the POP that is geographically closest to the user.</span></span>
2. <span data-ttu-id="cabb8-118">Als het bestand niet beschikbaar is in het cachegeheugen van de randservers in het POP, vraagt de randserver het bestand aan bij de oorsprong.</span><span class="sxs-lookup"><span data-stu-id="cabb8-118">If the edge servers in the POP do not have the file in their cache, the edge server requests the file from the origin.</span></span>  <span data-ttu-id="cabb8-119">De oorsprong kan Azure-web-app, Azure Cloud-service, Azure Storage-account of een openbaar toegankelijke webserver zijn.</span><span class="sxs-lookup"><span data-stu-id="cabb8-119">The origin can be an Azure Web App, Azure Cloud Service, Azure Storage account, or any publicly accessible web server.</span></span>
3. <span data-ttu-id="cabb8-120">De oorsprong retourneert het bestand naar de randserver, inclusief optionele HTTP-headers met een beschrijving van de TTL (Time-to-Live) van het bestand.</span><span class="sxs-lookup"><span data-stu-id="cabb8-120">The origin returns the file to the edge server, including optional HTTP headers describing the file's Time-to-Live (TTL).</span></span>
4. <span data-ttu-id="cabb8-121">De randserver neemt het bestand op in de cache en retourneert het bestand naar de oorspronkelijke aanvrager (Alice).</span><span class="sxs-lookup"><span data-stu-id="cabb8-121">The edge server caches the file and returns the file to the original requestor (Alice).</span></span>  <span data-ttu-id="cabb8-122">Het bestand blijft in cache op de randserver totdat de TTL verloopt.</span><span class="sxs-lookup"><span data-stu-id="cabb8-122">The file remains cached on the edge server until the TTL expires.</span></span>  <span data-ttu-id="cabb8-123">Als de oorsprong geen TTL heeft opgegeven, is de standaard-TTL zeven dagen.</span><span class="sxs-lookup"><span data-stu-id="cabb8-123">If the origin didn't specify a TTL, the default TTL is seven days.</span></span>
5. <span data-ttu-id="cabb8-124">Extra gebruikers kunnen dan diezelfde URL gebruiken om hetzelfde bestand aan te vragen en worden mogelijk ook omgeleid naar hetzelfde POP.</span><span class="sxs-lookup"><span data-stu-id="cabb8-124">Additional users may then request the same file using that same URL, and may also be directed to that same POP.</span></span>
6. <span data-ttu-id="cabb8-125">Als de TTL voor het bestand niet is verlopen, retourneert de randserver het bestand uit de cache.</span><span class="sxs-lookup"><span data-stu-id="cabb8-125">If the TTL for the file hasn't expired, the edge server returns the file from the cache.</span></span>  <span data-ttu-id="cabb8-126">Dit resulteert in een snellere, responsievere gebruikerservaring.</span><span class="sxs-lookup"><span data-stu-id="cabb8-126">This results in a faster, more responsive user experience.</span></span>
