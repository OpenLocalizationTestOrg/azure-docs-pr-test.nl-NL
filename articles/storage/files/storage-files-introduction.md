## <a name="why-azure-file-storage-is-useful"></a><span data-ttu-id="e328f-112">Waarom Azure File Storage handig is</span><span class="sxs-lookup"><span data-stu-id="e328f-112">Why Azure File storage is useful</span></span>

<span data-ttu-id="e328f-113">Azure File Storage biedt u de mogelijkheid om Windows Server-, Linux-, of NAS-bestandsservers die lokaal of in de cloud worden gehost te vervangen door een besturingssysteemvrije cloudbestandsshare.</span><span class="sxs-lookup"><span data-stu-id="e328f-113">Azure File storage allows you to replace Windows Server, Linux, or NAS-based file servers hosted on-premises or in the cloud with an OS-free cloud file share.</span></span> <span data-ttu-id="e328f-114">Azure File Storage biedt de volgende voordelen:</span><span class="sxs-lookup"><span data-stu-id="e328f-114">Azure File storage has the following benefits:</span></span>

* <span data-ttu-id="e328f-115">**Gedeelde toegang** Azure-bestandsshares bieden ondersteuning voor de industrienorm SMB-protocol, wat betekent dat u uw lokale bestandsshares naadloos kunt vervangen door de Azure-bestandsshares zonder dat u zich zorgen hoeft te maken over de compatibiliteit van toepassingen.</span><span class="sxs-lookup"><span data-stu-id="e328f-115">**Shared access** Azure File shares support the industry standard SMB protocol, meaning you can seamlessly replace your on-premises file shares with Azure File shares without worrying about application compatibility.</span></span> <span data-ttu-id="e328f-116">De mogelijkheid om een bestandsshare te gebruiken vanaf verschillende computers en vanuit diverse toepassingen/exemplaren, is een belangrijk voordeel van Azure File Storage.</span><span class="sxs-lookup"><span data-stu-id="e328f-116">Being able to access a file share from multiple machines and applications/instances is a significant advantage with Azure File storage.</span></span>

* <span data-ttu-id="e328f-117">**Volledig beheerd** Azure-bestandsshares kunnen worden gemaakt zonder dat het nodig is om hardware of een besturingssysteem te beheren. Dit betekent dat u zich geen zorgen hoeft te maken over het patchen van de OS-server met essentiële beveiligingsupgrades of het vervangen van defecte harde schijven.</span><span class="sxs-lookup"><span data-stu-id="e328f-117">**Fully Managed** Azure File shares can be created without the need to manage hardware or an OS, which means you don't have to deal with patching the server OS with critical security upgrades or replacing faulty hard disks.</span></span>

* <span data-ttu-id="e328f-118">**Scripts en hulpprogramma's** PowerShell-cmdlets en de Azure-CLI zijn beschikbaar voor het maken, koppelen en beheren van Azure-bestandsshares als onderdeel van het beheer van Azure-toepassingen.</span><span class="sxs-lookup"><span data-stu-id="e328f-118">**Scripting and Tooling** PowerShell cmdlets and Azure CLI can be used to create, mount, and manage Azure File shares as part of the administration of Azure applications.</span></span> <span data-ttu-id="e328f-119">U kunt Azure-bestandsshares maken en beheren met behulp van [Azure Portal](https://portal.azure.com) en [Azure Storage Explorer](https://storageexplorer.com).</span><span class="sxs-lookup"><span data-stu-id="e328f-119">You can create and manage Azure File shares using the [Azure portal](https://portal.azure.com) and the [Azure Storage Explorer](https://storageexplorer.com).</span></span> 

* <span data-ttu-id="e328f-120">**Robuust** Azure File Storage is vanaf de grond opgebouwd om altijd beschikbaar te zijn.</span><span class="sxs-lookup"><span data-stu-id="e328f-120">**Resiliency** Azure File storage has been built from the ground up to be always available.</span></span> <span data-ttu-id="e328f-121">Het vervangen van on-premises bestandsshares door Azure File Storage betekent dat u niet langer problemen ondervindt door lokale stroomstoringen of netwerkstoringen.</span><span class="sxs-lookup"><span data-stu-id="e328f-121">Replacing on-premises file shares with Azure File storage means you no longer have to wake up to deal with local power outages or network issues.</span></span> 

* <span data-ttu-id="e328f-122">**Vertrouwd programmeren** Toepassingen die worden uitgevoerd in Azure hebben toegang tot de gegevens in de share via [API's voor I/O van het bestandssysteem](https://msdn.microsoft.com/library/system.io.file.aspx).</span><span class="sxs-lookup"><span data-stu-id="e328f-122">**Familiar Programmability** Applications running in Azure can access data on the share via [file system I/O APIs](https://msdn.microsoft.com/library/system.io.file.aspx).</span></span> <span data-ttu-id="e328f-123">Dat betekent dat ontwikkelaars bestaande code en vaardigheden kunnen inzetten voor het migreren van bestaande toepassingen.</span><span class="sxs-lookup"><span data-stu-id="e328f-123">Developers can therefore leverage their existing code and skills to migrate existing applications.</span></span> <span data-ttu-id="e328f-124">Naast API's voor I/O van het bestandssysteem kunt u elke Azure Storage-clientbibliotheek gebruiken, zoals die voor [.NET](/dotnet/api/overview/azure/storage?view=azure-dotnet), of de [REST-API van Azure Storage](/rest/api/storageservices/file-service-rest-api).</span><span class="sxs-lookup"><span data-stu-id="e328f-124">In addition to System IO APIs, you can use any of the Azure storage client Libraries, such as the one for [.NET](/dotnet/api/overview/azure/storage?view=azure-dotnet), or the [Azure Storage REST API](/rest/api/storageservices/file-service-rest-api).</span></span>

<span data-ttu-id="e328f-125">Azure-bestandsshares kunnen worden gebruikt voor het volgende:</span><span class="sxs-lookup"><span data-stu-id="e328f-125">Azure File shares can be used to:</span></span>

* <span data-ttu-id="e328f-126">**On-premises bestandsservers vervangen** Azure File Storage kan worden gebruikt om bestandsshares op traditionele on-premises bestandsservers of NAS-apparaten volledig te vervangen.</span><span class="sxs-lookup"><span data-stu-id="e328f-126">**Replace on-premises file servers** Azure File storage can be used to completely replace file shares on traditional on-premises file servers or NAS devices.</span></span> <span data-ttu-id="e328f-127">Populaire besturingssystemen, zoals Windows, Mac OS en Linux kunnen gemakkelijk een Azure-bestandsshare koppelen, waar ter wereld ze zich maar bevinden.</span><span class="sxs-lookup"><span data-stu-id="e328f-127">Popular operating systems such as Windows, macOS, and Linux can easily mount an Azure File share wherever they are in the world.</span></span>

    <span data-ttu-id="e328f-129">Met Azure File Storage kunt u toepassingen die gebruikmaken van on-premises bestandsshares gemakkelijk 'liften en shiften' om gegevens tussen onderdelen van de toepassing te delen.</span><span class="sxs-lookup"><span data-stu-id="e328f-129">Azure File storage makes it easy to "lift and shift" applications to the cloud that use on-premises file shares to share data between parts of the application.</span></span> <span data-ttu-id="e328f-130">Om dit te realiseren, maakt elke virtuele machine verbinding met de bestandsshare en kan deze vervolgens bestanden lezen en schrijven op dezelfde manier als dit zou gebeuren op een on-premises bestandsshare.</span><span class="sxs-lookup"><span data-stu-id="e328f-130">To implement this, each VM connects to the file share and then it can read and write files just like it would against an on-premises file share.</span></span>

    * <span data-ttu-id="e328f-133">**Gedeelde toepassingsinstellingen**</span><span class="sxs-lookup"><span data-stu-id="e328f-133">**Shared Application Settings**</span></span>
    
        <span data-ttu-id="e328f-134">Een algemene patroon voor gedistribueerde toepassingen is de aanwezigheid van configuratiebestanden op een centrale locatie waar ze toegankelijk zijn vanuit veel andere VM's.</span><span class="sxs-lookup"><span data-stu-id="e328f-134">A common pattern for distributed applications is to have configuration files in a centralized location where they can be accessed from many different VMs.</span></span> <span data-ttu-id="e328f-135">Dergelijke configuratiebestanden kunnen nu worden opgeslagen op een Azure-bestandsshare en worden gelezen door alle exemplaren van een toepassing.</span><span class="sxs-lookup"><span data-stu-id="e328f-135">Such configuration files can now be stored in an Azure File share, and read by all application instances.</span></span> <span data-ttu-id="e328f-136">Deze instellingen kunnen ook worden beheerd via de REST-interface waarmee u wereldwijd toegang krijgt tot de configuratiebestanden.</span><span class="sxs-lookup"><span data-stu-id="e328f-136">These settings can also be managed via the REST interface, which allows worldwide access to the configuration files.</span></span>
