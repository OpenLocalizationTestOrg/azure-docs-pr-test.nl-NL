# <a name="simulate-failures-during-service-workloads"></a>Storingen simuleren tijdens servicewerkbelastingen

Er zijn scenario's, echter, waar een expliciete interleaving van de werkbelasting van de client en storingen mogelijk nodig zijn. Hallo van client werkbelasting en fouten interleaving zorgt ervoor dat service Hallo daadwerkelijk wordt een bepaalde actie bij fout gebeurt. Beheerniveau Hallo die testbaarheid biedt, kunnen deze worden op nauwkeurige punten van Hallo kunnen worden uitgevoerd. Dit doen ontstaan van fouten in verschillende statussen in de toepassing hello kan vinden van de fouten en verbeteren.

    private static async Task RunFaultAsync(Uri applicationName, ServiceFabricFaults fault, ReplicaSelector selector, FabricClient client)

            case ServiceFabricFaults.RestartCodePackage:
                await client.ApplicationManager.RestartDeployedCodePackageAsync(applicationName, selector, CompletionMode.Verify);
                break;
            case ServiceFabricFaults.RemoveReplica:
                await client.ServiceManager.RemoveReplicaAsync(selector, CompletionMode.Verify, false);
                break;
            case ServiceFabricFaults.MovePrimary:
                await client.ServiceManager.MovePrimaryAsync(selector.PartitionSelector);
                break;
        }
