## <a name="example"></a><span data-ttu-id="c61cf-115">Voorbeeld</span><span class="sxs-lookup"><span data-stu-id="c61cf-115">Example</span></span>
<span data-ttu-id="c61cf-116">De volgende code ziet u het gebruik van meerdere shard uitvoeren van query's met behulp van een bepaalde **ShardMap** met de naam *myShardMap*.</span><span class="sxs-lookup"><span data-stu-id="c61cf-116">The following code illustrates the usage of multi-shard querying using a given **ShardMap** named *myShardMap*.</span></span> 

    using (MultiShardConnection conn = new MultiShardConnection( 
                                        myShardMap.GetShards(), 
                                        myShardConnectionString) 
          ) 
    { 
    using (MultiShardCommand cmd = conn.CreateCommand())
           { 
            cmd.CommandText = "SELECT c1, c2, c3 FROM ShardedTable"; 
            cmd.CommandType = CommandType.Text; 
            cmd.ExecutionOptions = MultiShardExecutionOptions.IncludeShardNameColumn; 
            cmd.ExecutionPolicy = MultiShardExecutionPolicy.PartialResults; 

            using (MultiShardDataReader sdr = cmd.ExecuteReader()) 
                { 
                    while (sdr.Read())
                        { 
                            var c1Field = sdr.GetString(0); 
                            var c2Field = sdr.GetFieldValue<int>(1); 
                            var c3Field = sdr.GetFieldValue<Int64>(2);
                        } 
                 } 
           } 
    } 


<span data-ttu-id="c61cf-117">Een belangrijk verschil is het maken van meerdere shard-verbindingen.</span><span class="sxs-lookup"><span data-stu-id="c61cf-117">A key difference is the construction of multi-shard connections.</span></span> <span data-ttu-id="c61cf-118">Waar **SqlConnection** is van invloed op een individuele database de **MultiShardConnection** duurt een ***verzameling shards*** als invoer.</span><span class="sxs-lookup"><span data-stu-id="c61cf-118">Where **SqlConnection** operates on a single database, the **MultiShardConnection** takes a ***collection of shards*** as its input.</span></span> <span data-ttu-id="c61cf-119">Vul de verzameling van shards uit een shard-toewijzing.</span><span class="sxs-lookup"><span data-stu-id="c61cf-119">Populate the collection of shards from a shard map.</span></span> <span data-ttu-id="c61cf-120">De query wordt vervolgens uitgevoerd op de verzameling van shards met **UNION ALL** semantiek voor het samenstellen van een enkele algemene resultaat.</span><span class="sxs-lookup"><span data-stu-id="c61cf-120">The query is then executed on the collection of shards using **UNION ALL** semantics to assemble a single overall result.</span></span> <span data-ttu-id="c61cf-121">U kunt desgewenst de naam van de shard waar de rij is afkomstig uit kan worden toegevoegd aan de uitvoer met behulp de **ExecutionOptions** eigenschap op de opdracht.</span><span class="sxs-lookup"><span data-stu-id="c61cf-121">Optionally, the name of the shard where the row originates from can be added to the output using the **ExecutionOptions** property on command.</span></span>
