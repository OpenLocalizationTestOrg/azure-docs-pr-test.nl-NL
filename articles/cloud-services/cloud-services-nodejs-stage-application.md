## <a name="step-1-stage-an-application"></a><span data-ttu-id="96089-108">Stap 1: Voorbereiden van een toepassing</span><span class="sxs-lookup"><span data-stu-id="96089-108">Step 1: Stage an Application</span></span>
<span data-ttu-id="96089-109">Deze taak wordt beschreven hoe een toepassing installeert met behulp van de **Microsoft Azure PowerShell**.</span><span class="sxs-lookup"><span data-stu-id="96089-109">This task covers how to stage an application by using the **Microsoft Azure PowerShell**.</span></span>

1. <span data-ttu-id="96089-110">Bij het publiceren van een service, geeft de **-sleuf** -parameter voor de **Publish-AzureServiceProject** cmdlet.</span><span class="sxs-lookup"><span data-stu-id="96089-110">When publishing a service, simply pass the **-Slot** parameter to the **Publish-AzureServiceProject** cmdlet.</span></span>
   
   ```powershell
   Publish-AzureServiceProject -Slot staging
   ```
2. <span data-ttu-id="96089-111">Meld u aan bij de [klassieke Azure-portal] en selecteer **Cloudservices**.</span><span class="sxs-lookup"><span data-stu-id="96089-111">Log on to the [Azure classic portal] and select **Cloud Services**.</span></span> <span data-ttu-id="96089-112">Nadat de cloud service wordt gemaakt en de **fasering** kolom status is bijgewerkt naar **met**, klikt u op de naam van de service.</span><span class="sxs-lookup"><span data-stu-id="96089-112">After the cloud service is created and the **Staging** column status has been updated to **Running**, click on the service name.</span></span>
   
   ![weergeven van een actieve service portal][cloud-service]
3. <span data-ttu-id="96089-114">Selecteer de **Dashboard**, en selecteer vervolgens **fasering**.</span><span class="sxs-lookup"><span data-stu-id="96089-114">Select the **Dashboard**, and then select **Staging**.</span></span>
   
   ![cloud service-dashboard][cloud-service-dashboard]
4. <span data-ttu-id="96089-116">Noteer de waarde in de **Site-URL** vermelding aan de rechterkant.</span><span class="sxs-lookup"><span data-stu-id="96089-116">Note the value in the **Site URL** entry to the right.</span></span> <span data-ttu-id="96089-117">De DNS-naam is een verborgen interne ID die Azure gegenereerd.</span><span class="sxs-lookup"><span data-stu-id="96089-117">The DNS name is an obfuscated internal ID that Azure generated.</span></span>
   
    ![site-url][cloud-service-staging-url]

<span data-ttu-id="96089-119">Nu kunt u controleren of de toepassing correct in de testomgeving werkt met behulp van de staging-site-URL.</span><span class="sxs-lookup"><span data-stu-id="96089-119">Now you can verify that the application is working correctly in the staging environment by using the staging site URL.</span></span>

## <a name="step-2-upgrade-an-application-in-production-by-swapping-vips"></a><span data-ttu-id="96089-120">Stap 2: Een toepassing in productie door wisselen VIP's bijwerken</span><span class="sxs-lookup"><span data-stu-id="96089-120">Step 2: Upgrade an Application in Production by Swapping VIPs</span></span>
<span data-ttu-id="96089-121">Nadat u de bijgewerkte versie van een toepassing in de testomgeving hebt gecontroleerd, kunt u snel maken het beschikbaar in productie door wisselen van de virtuele IP-adressen (VIP's) van de fasering en productie-omgevingen.</span><span class="sxs-lookup"><span data-stu-id="96089-121">After you have verified the upgraded version of an application in the staging environment, you can quickly make it available in production by swapping the virtual IPs (VIPs) of the staging and production environments.</span></span>

> [!NOTE]
> <span data-ttu-id="96089-122">Deze stap wordt ervan uitgegaan dat u hebt al een toepassing naar productie geïmplementeerd en voorbereid de bijgewerkte versie van de toepassing.</span><span class="sxs-lookup"><span data-stu-id="96089-122">This step assumes that you have already deployed an application to production and staged the upgraded version of the application.</span></span>
> 
> 

1. <span data-ttu-id="96089-123">Meld u aan bij de [klassieke Azure-portal], klikt u op **Cloudservices** en selecteer vervolgens de naam van de service.</span><span class="sxs-lookup"><span data-stu-id="96089-123">Log into the [Azure classic portal], click **Cloud Services** and then select the service name.</span></span>
2. <span data-ttu-id="96089-124">Van de **Dashboard**, selecteer **fasering**, en klik vervolgens op **wisselen** aan de onderkant van de pagina.</span><span class="sxs-lookup"><span data-stu-id="96089-124">From the **Dashboard**, select **Staging**, and then click **Swap** at the bottom of the page.</span></span> <span data-ttu-id="96089-125">Hiermee opent u het dialoogvenster VIP's wisselen.</span><span class="sxs-lookup"><span data-stu-id="96089-125">This opens the VIP Swap dialog.</span></span>
   
   ![VIP wisselen dialoogvenster][vip-swap-dialog]
3. <span data-ttu-id="96089-127">Lees de informatie en klik vervolgens op **OK**.</span><span class="sxs-lookup"><span data-stu-id="96089-127">Review the information, and then click **OK**.</span></span> <span data-ttu-id="96089-128">De twee implementaties beginnen met het bijwerken van de staging implementatie schakelt over naar de productie en de productie-implementatie voor fasering verandert.</span><span class="sxs-lookup"><span data-stu-id="96089-128">The two deployments begin updating as the staging deployment switches to production and the production deployment switches to staging.</span></span>

<span data-ttu-id="96089-129">U hebt voorbereid, een implementatie en een productie-implementatie door het wisselen van VIP's met de implementatie op gefaseerde wijze bijgewerkt.</span><span class="sxs-lookup"><span data-stu-id="96089-129">You have successfully staged a deployment and upgraded a production deployment by swapping VIPs with the deployment in staging.</span></span>
