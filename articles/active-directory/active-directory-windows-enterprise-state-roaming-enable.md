## <a name="data-retention"></a><span data-ttu-id="93e63-132">Bewaartijd van gegevens</span><span class="sxs-lookup"><span data-stu-id="93e63-132">Data retention</span></span>
<span data-ttu-id="93e63-133">Gegevens die zijn gesynchroniseerd met Azure via Enterprise State Roaming blijven voor onbepaalde tijd bewaard tenzij een handmatige delete-bewerking wordt uitgevoerd of wordt bepaald dat de betrokken gegevens verlopen.</span><span class="sxs-lookup"><span data-stu-id="93e63-133">Data synced to Azure via Enterprise State Roaming will be retained indefinitely unless a manual delete operation is performed or the data in question is determined to be stale.</span></span> 

<span data-ttu-id="93e63-134">**Expliciet verwijderen:** de gegevens worden verwijderd wanneer u een Azure-beheerder verwijdert een gebruiker of een map of een beheerder aanvragen expliciet dat gegevens worden verwijderd.</span><span class="sxs-lookup"><span data-stu-id="93e63-134">**Explicit deletion:** The data is deleted when an Azure admin deletes a user or a directory or an admin requests explicitly that data is to be deleted.</span></span>

* <span data-ttu-id="93e63-135">**Verwijderen van een gebruiker**: wanneer een gebruiker wordt verwijderd in Azure AD, het gebruikersaccount zwervende wordt gemarkeerd voor verwijdering en tussen 90 tot 180 dagen wordt verwijderd.</span><span class="sxs-lookup"><span data-stu-id="93e63-135">**User deletion**: When a user is deleted in Azure AD, the user account roaming data will be marked for deletion and will be deleted between 90 to 180 days.</span></span>

* <span data-ttu-id="93e63-138">**Op aanvraag verwijdering**: als de beheerder van Azure AD wil Verwijder handmatig de gegevens of instellingsgegevens voor een specifieke gebruiker, de beheerder een ticket met kunt indienen [ondersteuning van Azure](https://azure.microsoft.com/support/).</span><span class="sxs-lookup"><span data-stu-id="93e63-138">**On request deletion**: If the Azure AD admin wants to manually delete a specific user’s data or settings data, the admin can file a ticket with [Azure support](https://azure.microsoft.com/support/).</span></span> 

<span data-ttu-id="93e63-139">**Verouderde gegevens verwijderen**: gegevens die niet is geopend voor één jaar ('de bewaarperiode'), worden behandeld als verouderd en kan worden verwijderd uit Azure.</span><span class="sxs-lookup"><span data-stu-id="93e63-139">**Stale data deletion**: Data that has not been accessed for one year (“the retention period”) will be treated as stale and may be deleted from Azure.</span></span> <span data-ttu-id="93e63-140">De bewaarperiode kan worden gewijzigd maar niet minder dan 90 dagen.</span><span class="sxs-lookup"><span data-stu-id="93e63-140">The retention period is subject to change but will not be less than 90 days.</span></span> <span data-ttu-id="93e63-141">Verouderde gegevens mogelijk een specifieke set van Windows-toepassing of alle instellingen voor een gebruiker.</span><span class="sxs-lookup"><span data-stu-id="93e63-141">The stale data may be a specific set of Windows/application settings or all settings for a user.</span></span> <span data-ttu-id="93e63-142">Bijvoorbeeld:</span><span class="sxs-lookup"><span data-stu-id="93e63-142">For example:</span></span>

* <span data-ttu-id="93e63-144">Als een gebruiker heeft de synchronisatie van instellingen op alle apparaten zijn uitgeschakeld, klikt u vervolgens geen van de instellingsgegevens wordt geopend en alle instellingsgegevens voor die gebruiker wordt zijn verouderd en kan worden verwijderd na de bewaarperiode.</span><span class="sxs-lookup"><span data-stu-id="93e63-144">If a user has turned off settings sync on all his/her devices, then none of the settings data will be accessed, and all the settings data for that user will become stale and may be deleted after the retention period.</span></span> 
* <span data-ttu-id="93e63-145">Als de Azure AD-directory-beheerder uitgeschakeld Enterprise State Roaming voor de hele map, worden alle gebruikers wordt in die map stopt met het synchroniseren van de instellingen en alle instellingsgegevens voor alle gebruikers worden verlopen en kan worden verwijderd na de bewaarperiode.</span><span class="sxs-lookup"><span data-stu-id="93e63-145">If the Azure AD directory admin turns off Enterprise State Roaming for the entire directory, then all users in that directory will stop syncing settings, and all settings data for all users will become stale and may be deleted after the retention period.</span></span> 

<span data-ttu-id="93e63-146">**Gegevensherstel verwijderd**: het beleid voor het bewaren van gegevens kan niet worden geconfigureerd.</span><span class="sxs-lookup"><span data-stu-id="93e63-146">**Deleted data recovery**: The data retention policy is not configurable.</span></span> <span data-ttu-id="93e63-147">Nadat de gegevens is nu definitief verwijderd, is deze niet worden hersteld.</span><span class="sxs-lookup"><span data-stu-id="93e63-147">Once the data has been permanently deleted, it will not be recoverable.</span></span> <span data-ttu-id="93e63-148">Het is echter belangrijk te weten dat de instellingsgegevens alleen worden verwijderd uit Azure, niet de apparaten van eindgebruikers.</span><span class="sxs-lookup"><span data-stu-id="93e63-148">However, it’s important to note that the settings data will only be deleted from Azure, not the end-user device.</span></span> <span data-ttu-id="93e63-149">Als een apparaat later opnieuw verbinding mee met de service Enterprise State Roaming maakt, worden de instellingen opnieuw gesynchroniseerd en opgeslagen in Azure.</span><span class="sxs-lookup"><span data-stu-id="93e63-149">If any device later reconnects to the Enterprise State Roaming service, the settings will again be synced and stored in Azure.</span></span>
